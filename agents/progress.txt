## Codebase Patterns
- Scaffolded via `npx sv create . --template minimal --types ts --no-dir-check --no-download-check --install npm --add prettier eslint tailwindcss="plugins:none" sveltekit-adapter="adapter:static"` â€” the `sv` CLI's non-interactive mode requires every multiselect add-on option to be explicitly set (e.g. `tailwindcss="plugins:none"`) or it still prompts.
- This SvelteKit/Vite version has no separate `svelte.config.js` â€” the adapter and Svelte compiler options are configured inline in `vite.config.ts` via the `sveltekit({ adapter: adapter(), compilerOptions: {...} })` plugin call.
- Static prerendering is enabled via `src/routes/+layout.ts` exporting `export const prerender = true;` (applies to the whole route tree since this is a single-page app).
- Tailwind CSS v4 is wired via the `@tailwindcss/vite` plugin (not a `tailwind.config.js` + PostCSS pipeline) â€” theme customization for docs/01-design-system.md's tokens should go through Tailwind v4's CSS-based `@theme` config, not a `tailwind.config.ts` `theme.extend` block (that's the v3 pattern the PRD assumed; confirm the v4 equivalent when doing US-002).
- `npm run check` = typecheck (`svelte-kit sync && svelte-check`), `npm run lint` = `prettier --check . && eslint .`, `npm run build` = `vite build` (outputs static site to `build/`).
- `.prettierignore` excludes `/docs/`, `/agents/`, and `/.claude/` â€” these are hand-authored project docs/agent files and local tool config, not app source; Prettier's default file walk would otherwise reformat and reflow the markdown/JSON prose there.
- Design tokens live in an `@theme { ... }` block inside `src/routes/layout.css` (imported once via the root layout). Tailwind v4 auto-generates a `:root, :host { --color-base: ...; --font-mono: ...; }` block in the compiled CSS from every `@theme` entry â€” no need to hand-write a separate mirrored `:root` block for JS/canvas consumers, `var(--color-*)` and `var(--font-*)` are already real custom properties.
- Color/font utility class names follow Tailwind's `--color-<name>` / `--font-<name>` â†’ `<utility>-<name>` mapping directly: `--color-green-bright` â†’ `text-green-bright`/`bg-green-bright`, `--color-text-muted` â†’ `text-text-muted` (note the doubled "text"), `--font-mono` â†’ `font-mono`.
- Fonts are self-hosted via `@fontsource-variable/jetbrains-mono` and `@fontsource-variable/inter` (npm packages, imported via `@import` in `layout.css`) rather than manually sourced font files â€” these already ship variable-weight woff2 subset by `unicode-range` with `font-display: swap` built in.
- No border-radius config override was needed for "hard edges everywhere" â€” Tailwind applies no radius unless a `rounded-*` utility is used, so the requirement is satisfied by omission, not a theme reset.

What doesn't reliably persist/apply is (a) directly assigning `window.scrollTo(...)` via `rodney js` right before an unrelated later command, and (b) the `screenshot -w/-h` viewport override for responsive/media-query testing â€” `window.innerWidth` can read the overridden value in a later `rodney js` call while `matchMedia('(min-width: ...)')` still evaluates against the un-overridden viewport, so don't trust breakpoint-dependent `rodney js` assertions after a screenshot resize; trust the screenshot's own pixel output instead, and drive responsive interactions (like clicking a mobile-only button) via `element.click()` in JS rather than `rodney click` (which requires the element to pass a visibility/hit-test check that may not match the emulated width).

- **Learnings for future iterations:**
  - Don't put a marker character + trailing space directly in template text when you need that space preserved (e.g. `<span>{'> '}</span>`) â€” Svelte's compiler trims it, and `svelte/no-useless-mustaches` also flags a mustache wrapping a plain string literal. Use CSS spacing (margin/padding) on the marker element instead.
  - For any future non-color state indicator (e.g. cert "expiring soon" badges in US-012), use `aria-current`/`aria-selected`/similar plus an `aria-hidden` decorative marker â€” don't let a visual-only prefix/suffix leak into the accessible name.
